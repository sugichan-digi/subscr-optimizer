SubsOptimizer API (1.0.0)

Download OpenAPI specification:

SubsOptimizer（サブスクリプション最適化ツール）のバックエンド API 仕様書です。

認証について

/subscriptions 系エンドポイントと /auth/logout、/auth/account は Bearer トークン認証が必要です。

ログイン・新規登録・ゲストログインのいずれかで取得したトークンを リクエストの Authorization ヘッダーに付与してください。

Authorization: Bearer {token}

トークンの有効期限は 30日間 です。

開発モードについて

本 API はモック実装です。データはサーバー上の JSON ファイルに保存されます。 POST /auth/forgot-password はメール送信を行わず、代わりにレスポンス内の dev_reset_token フィールドにトークンを返します。 本番環境に移行する際はこのフィールドを削除し、メール送信処理を実装してください。

ゲストログイン

メールアドレス・パスワードなしでゲストアカウントを作成しログインします。 リクエストボディは不要です（空のオブジェクトでも可）。

注意事項:

  • ゲストアカウントへの通常アカウント移行機能はありません
  • 退会するとすべてのデータが削除されます
  • ゲストユーザーの退会時はパスワード確認が不要です

パスワードリセットトークンの発行

登録済みのメールアドレスに対してパスワードリセット用トークンを発行します。 トークンの有効期限は 1時間 です。

セキュリティ: メールアドレスが未登録の場合も同じレスポンスを返します （ユーザー列挙攻撃への対策）。

開発モード: 実際のメール送信は行われず、レスポンスの dev_reset_token フィールドにトークンが含まれます。 本番環境ではこのフィールドを削除し、メール送信処理を実装してください。

アカウント退会（全データ削除）

ログイン中のアカウントを退会・削除します。

この操作は取り消せません。 以下のデータがすべて削除されます：

  • ユーザー情報
  • 全セッション
  • パスワードリセットトークン
  • 登録済みサブスクリプションデータ

ゲストユーザー: password フィールドは不要です（空のボディでも可）。 通常ユーザー: 現在のパスワードが必須です。

Authorizations:
BearerAuth
